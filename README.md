# pulsefeed

PulseFeed is a static frontend app that reads `feed.json` and renders planning updates in the browser. The feed contents are generated by the Python scraper in `scraper/scrape.py`.

## Local development

1. Install scraper dependencies:
   ```bash
   python3 -m pip install -r scraper/requirements.txt
   ```
2. Run the scraper manually:
   ```bash
   python3 scraper/scrape.py
   ```
3. Preview the static app locally with a simple web server:
   ```bash
   python3 -m http.server 8000
   ```
4. Open `http://localhost:8000` in your browser.

## Architecture

- `index.html` — static user interface
- `styles.css` — styles for the static app
- `app.js` — client-side interactivity, fetches `/feed.json`
- `feed.json` — feed data generated by the scraper
- `scraper/scrape.py` — Python scraper that writes `feed.json`
- `.github/workflows/scrape.yml` — daily scraper workflow

## Azure hosting options

### Recommended: Azure Static Web Apps

This is the most cost-effective hosting option for the static site.

- App location: `/`
- API location: leave empty
- Output location: leave empty

Because the site is fully static, Azure Static Web Apps can serve `index.html`, `app.js`, `styles.css`, and `feed.json` directly.

### Alternative: Azure Storage static website

Use Azure Storage static website hosting for the same static files. Upload the repository root files to the storage account and enable static website support.

## Scheduled feed generation

The GitHub Actions workflow at `.github/workflows/scrape.yml` runs the scraper daily and updates `feed.json` in the repository.

### Optional Azure Blob publish

If you want the feed to be pushed to an Azure storage container or CDN, configure the following repository secrets:

- `AZURE_STORAGE_ACCOUNT`
- `AZURE_STORAGE_KEY`
- `AZURE_STORAGE_CONTAINER`

When these secrets are set, the workflow uploads `feed.json` to the configured Azure Blob container after scraping.

## Manual feed update

Run the scraper manually anytime:
